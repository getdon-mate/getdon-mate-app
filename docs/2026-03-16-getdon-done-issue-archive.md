# Getdon Done Issue Archive

## 목적

`Getdon` 팀의 완료 이슈를 Linear에서 정리하기 전에, 중복되거나 세분화된 완료 이슈를 상위 흐름 기준으로 다시 묶어 남긴 아카이브 문서다.  
이 문서는 "무엇이 끝났는가"보다 "어떤 순서로 제품이 만들어졌는가"를 추적하기 쉽게 만드는 데 목적이 있다.

## 정리 원칙

- 상위-하위 관계가 있는 이슈는 상위 트랙 아래로 묶었다.
- 같은 작업 흐름에서 분리된 `FE / QA / Design / Content` 보조 이슈는 하나의 묶음으로 합쳤다.
- 완료 이슈가 지나치게 잘게 쪼개진 경우, 대표 결과 중심으로 요약하고 관련 이슈 ID를 함께 남겼다.
- 이후 Linear에서 완료 이슈를 비울 때는 이 문서를 단일 보존 기록으로 사용한다.

## 6. Linear 정리 권장 순서

도구 제약상 자동 삭제가 가능하다면 아래 순서가 가장 안전하다.

1. 단일 실행 성격의 leaf 완료 이슈 삭제
2. 같은 흐름의 QA / Design / Content 보조 이슈 삭제
3. 대표 기능 이슈 삭제
4. 마지막에 상위 트랙 이슈(`GET-12`, `GET-39`, `GET-1`, `GET-2`) 삭제

상위 이슈를 먼저 지우면 이후 완료 이슈의 맥락을 복원하기 어려워진다.  
이 문서는 그 맥락 손실을 막기 위한 최종 보존본이다.
