# GET-36 네비게이션 가드 및 회귀 검증 시나리오

## 1. 네비게이션 가드 규칙

### 1.1 인증 상태 가드

- `currentUser === null`인 상태에서는 `Login`만 접근 가능하다.
- 인증 해제(로그아웃/탈퇴/데모 데이터 초기화) 시 현재 화면과 무관하게 스택을 `Login` 단일 라우트로 재설정한다.
- 인증 성공(로그인/회원가입) 시 `Login` 히스토리를 제거하고 `AccountList`를 루트로 재설정한다.

### 1.2 상세 화면 진입 가드

- `AccountDetail` 진입 전 `selectedAccountId`가 반드시 설정되어야 한다.
- `selectedAccountId`가 `accounts`에 존재하지 않으면 `AccountDetail` 접근을 차단하고 `AccountList`로 이동시킨다.
- 상세 화면에서 계좌 삭제로 선택 계좌가 사라지면 자동으로 `AccountList`로 이탈한다.

### 1.3 back 동작 가드

- `AccountList -> AccountDetail`은 `navigate`로 push한다.
- `AccountDetail` 상단 back 버튼 및 시스템 back은 `AccountList`로 복귀한다.
- 상세에서 목록으로 복귀 시 `selectedAccountId`를 비워 stale selection을 방지한다.

## 2. 수동 QA 회귀 체크리스트

검증 기준: 각 항목은 실제 디바이스 또는 웹에서 1회 이상 수행하고 결과를 기록한다.

1. 앱 첫 진입 시 로그인 화면이 노출된다.
2. 올바른 계정으로 로그인하면 목록으로 이동하고, back으로 로그인 화면에 되돌아가지 않는다.
3. 회원가입 성공 시 목록으로 이동하고 신규 계정 상태가 유지된다.
4. 로그인 실패 시 현재 로그인 화면에 머물고 오류 메시지가 노출된다.
5. 목록에서 계좌 카드를 누르면 상세 화면으로 이동한다.
6. 상세에서 상단 back 버튼을 누르면 목록으로 복귀한다.
7. 상세에서 시스템 back(Android/web history)을 수행하면 목록으로 복귀한다.
8. 상세 진입 후 해당 계좌를 삭제하면 목록으로 자동 복귀하고 삭제된 계좌가 보이지 않는다.
9. 목록에서 로그아웃하면 로그인 화면으로 이동하고 인증 상태가 제거된다.
10. 목록에서 회원 탈퇴하면 로그인 화면으로 이동하고 재로그인이 불가능(삭제 계정)하다.
11. 데모 데이터 초기화를 실행하면 로그인 화면으로 이동하고 상태가 초기 mock 기준으로 복구된다.
12. 선택 계좌 없이 상세 화면 경로 상태가 만들어질 때(선택값 null/유효하지 않음) 목록으로 강제 이동된다.

## 3. 재사용 지침

- GET-12 계열 네비게이션 변경 시 본 문서 체크리스트 1~12를 최소 회귀 세트로 사용한다.
- 신규 라우트 추가 시 다음 항목을 먼저 갱신한다.
  - 인증 필요 여부
  - 진입 전 필수 상태(예: selected entity)
  - back 시 기대 복귀 화면
