# GET-24 [FE/BE][Spec] 계약 동기화 및 착수 게이트 확정

## 목적

- FE 화면 요구사항(GET-22)과 BE 계약(API/도메인) 간 매핑을 고정한다.
- 구현 착수 전 필수 합의 항목과 오픈 이슈를 분리해 리스크를 줄인다.
- 착수 가능 여부를 체크리스트로 판단할 수 있게 한다.

## 1) 동기화 범위

- 대상 도메인:
  - 인증(로그인/회원가입)
  - 모임통장 목록/상세
  - 월 회비/1회성 회비
  - 거래내역
  - 멤버
- 산출물:
  - 화면-API 계약 매핑표
  - 결정사항/오픈이슈 목록
  - 착수 게이트 체크리스트

## 2) 화면-API 계약 매핑 (초안)

## 인증

- FE 필요 입력:
  - `name`(회원가입), `email`, `password`
- 계약 합의 필요:
  - 중복 이메일 에러 코드/메시지
  - 로그인 실패 에러 코드 표준화

## 계좌 목록

- FE 필요 필드:
  - `id`, `groupName`, `bankName`, `balance`
  - `membersCount` 또는 `members[]`
  - `paidCount(currentMonth)` 또는 `duesRecords[]`
- 계약 합의 필요:
  - 목록 응답에 요약 파생값 포함 여부
  - 기본 정렬 기준(생성일/수정일)

## 계좌 상세

- FE 필요 필드:
  - `accountNumber`, `monthlyDuesAmount`, `dueDay`
  - `members[]`, `duesRecords[]`, `transactions[]`
  - `autoTransfer`, `oneTimeDues[]`
- 계약 합의 필요:
  - 상세 단건 API에서 포함할 하위 컬렉션 범위
  - 탭별 분리 API 필요 여부

## 회비

- FE 필요 필드:
  - `memberId`, `month`, `status`, `paidDate`, `amount`
- 계약 합의 필요:
  - 월 이동 시 조회 단위(월 단건/기간)
  - 상태 토글 API idempotency

## 거래

- FE 필요 필드:
  - `id`, `type`, `amount`, `description`, `date`, `category`, `memberId`
- 계약 합의 필요:
  - 필터링(전체/입금/출금)을 FE 처리할지 BE 쿼리로 받을지
  - 페이지네이션 기준(날짜/커서)

## 1회성 회비

- FE 필요 필드:
  - `id`, `title`, `amount`, `dueDate`, `records[]`
- 계약 합의 필요:
  - 월 회비와 도메인 분리 여부
  - 멤버별 상태 토글의 동시성 처리

## 3) 결정사항 기록

### 확정

- FE 화면 계약 기준 문서는 `GET-22`를 source of truth로 유지한다.
- 파생 계산은 selector에서 처리하고, API는 원본 데이터 중심으로 설계한다.
- FE는 빈 상태/오류 상태를 화면 레벨에서 명시적으로 렌더링한다.

### 미확정 (오픈 이슈)

- [ ] 목록 응답에 `paidCount`를 포함할지 여부
- [ ] 회비/거래 조회의 페이지네이션 규칙
- [ ] 자동이체/1회성 회비 API의 트랜잭션 경계
- [ ] 에러 코드 네이밍 컨벤션(공통 표준)

## 4) 착수 게이트 체크리스트

- [x] 화면 IA/유저 플로우 정의 완료 (`GET-21`)
- [x] 화면별 데이터 요구사항 정의 완료 (`GET-22`)
- [x] FE 상태/컴포넌트 분리 초안 정의 완료 (`GET-23` 문서)
- [ ] FE/BE 동기화 회의(또는 비동기 합의) 1회 완료
- [ ] 오픈 이슈에 대한 담당자/마감일 지정
- [ ] API 계약서(엔드포인트/필드/에러코드) 초안 확정
- [ ] 착수 승인 코멘트 기록

## 5) 실행 순서 제안

1. 오픈 이슈 우선순위 지정 (`P0/P1/P2`)
2. P0 이슈 합의 및 API 계약서 초안 잠금
3. FE 구현 착수 게이트 재점검
4. 게이트 통과 시 구현 이슈를 `In Progress` 전환

## 6) GET-24 산출물 요약

- 본 문서는 FE/BE 계약 동기화 범위, 결정사항, 오픈 이슈, 착수 게이트를 한 곳에 정리한 기준 문서다.
- 구현 착수 전 남은 항목은 `동기화 합의 1회 완료`와 `오픈 이슈 담당/기한 확정`이다.
