# Transaction Action Density Design

## Goal

거래 탭의 카드별 액션 과밀도를 줄이고, 멤버/게시판과 같은 메뉴 패턴으로 통일한다.

## Recommended Approach

1. 거래 카드 하단의 항상 보이는 `수정`, `삭제` 버튼을 제거한다.
2. 각 거래 카드 우측에 `...` 메뉴 버튼을 배치한다.
3. 메뉴 안에 `수정`, `삭제`를 넣고 `삭제`는 빨간 톤으로 분리한다.
4. 수정은 기존처럼 현재 거래 탭의 인라인 편집 폼으로 연결한다.
5. 삭제는 기존 확인 모달과 토스트를 유지한다.

## UX Notes

- 카드 본문은 거래 내용과 금액만 먼저 읽히게 유지한다.
- 메뉴 버튼은 멤버 카드와 동일한 크기와 원형 버튼 언어를 재사용한다.
- 메뉴는 한 카드 안에서만 열리고, 항목 선택 후 즉시 닫힌다.
- e2e 동선도 `거래 메뉴 열기 -> 수정/삭제`로 맞춘다.

## Error Handling

- `isMutating`일 때 메뉴 항목은 비활성화한다.
- 삭제 확인이 취소되면 아무 상태도 바꾸지 않는다.

## Testing

- 단위 테스트: 거래 메뉴 노출, 수정/삭제 실행, 기존 토스트 문구 유지
- e2e: 거래 수정 후 삭제 흐름을 메뉴 기반으로 재검증
