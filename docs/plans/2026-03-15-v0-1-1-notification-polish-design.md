# Getdon Mate v0.1.1 Notification And Density Polish Design

## Goal

운영 화면 중 가장 자주 보는 알림센터와 캘린더의 가독성을 높이고, 작은 화면에서 상세 탭/카드 밀도를 더 다듬는다. 기존 기능은 유지하고, 정보 정리와 탐색 속도를 높이는 방향으로 polish한다.

## Scope

- 알림센터에 간단한 필터(`전체`, `읽지 않음`, `안내`, `공지`) 추가
- 알림 카드의 유형 구분과 작은 화면 액션 정리
- 캘린더 일정 카드에 tone badge와 선택 날짜 헤더 추가
- 상세 탭 바 pill 폭과 패딩을 소폭 줄여 모바일 밀도 개선
- 관련 unit/e2e 회귀 추가

## Product Direction

알림센터는 “쌓인 카드 목록”이 아니라 빠르게 훑고 정리하는 작업 공간처럼 보여야 한다. 캘린더는 날짜 선택 후 리스트가 같은 정보 밀도로 반복되지 않도록, 일정 종류를 즉시 구분할 수 있게 한다.

## UX Decisions

### 1. Notification Filters

- 필터는 화면 상단 설명 아래 칩 형태로 둔다.
- 기본값은 `전체`.
- `읽지 않음`은 unread true만 노출한다.
- `안내`는 납부 안내/송금 요청 같은 운영 리마인드 알림을 묶는다.
- `공지`는 게시판/공지 계열을 묶는다.

### 2. Notification Card Density

- 카드 액션은 unread일 때만 보이되, 가로 폭을 줄이고 메타 정보를 정리한다.
- 작은 화면에서는 상단 액션 버튼도 덜 넓게 조정한다.
- 비어 있을 때는 복원 액션만 남기고 설명을 짧게 줄인다.

### 3. Calendar Tone

- 선택 날짜 섹션에 `YYYY년 M월 D일` 헤더를 넣는다.
- 일정 tone별로 작은 배지 텍스트를 둔다.
  - `회비`
  - `거래`
  - `공지`
- 캘린더 그리드 자체는 단순하게 유지하고, 종류 구분은 리스트에서 강하게 준다.

### 4. Mobile Density

- 상세 탭 pill의 최소 폭과 horizontal padding을 조금 줄인다.
- 알림/캘린더 카드 패딩도 과하지 않게 압축한다.
- 기능을 숨기지 않고, 세로 스크롤 길이만 줄이는 방향으로 조정한다.

## Testing

- unit: 알림 필터 동작, 캘린더 tone 표시, 탭 바 밀도 관련 노출
- e2e: 알림센터에서 리마인드 알림 필터링, 공지/안내 확인 흐름 유지
