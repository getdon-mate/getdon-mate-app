# Comment And Settings Density Design

## Goal

댓글 소유자 액션과 설정/알림 보조 카피를 줄여 화면 밀도를 더 단정하게 만든다.

## Recommended Approach

1. 댓글 수정/삭제를 항상 보이는 아이콘 두 개 대신 `...` 메뉴 안으로 이동한다.
2. 게시글 메뉴와 같은 시각 언어를 재사용한다.
3. 설정 화면의 설명 캡션은 제거하고 라벨만 남긴다.
4. 알림 화면 상단 보조 힌트는 제거해 제목, 카운트, 필터에만 집중시킨다.

## UX Notes

- 기능은 그대로 유지하고 노출 방식만 줄인다.
- 댓글 본문과 시간 정보가 먼저 읽히고, 액션은 필요할 때만 연다.
- 설정 화면은 계정 정보와 전역 액션만 보이는 간결한 목록으로 유지한다.

## Testing

- 단위 테스트: 댓글 메뉴 패턴, 설정 캡션 제거, 알림 상단 힌트 제거
- 전체 회귀: unit + web export + e2e
