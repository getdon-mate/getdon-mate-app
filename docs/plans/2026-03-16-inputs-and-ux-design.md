# Inputs And UX Design

**Date:** 2026-03-16

## Goal

입력 실수를 줄이는 공용 숫자 입력과 일자 선택 UI를 만들고, 알림/설정/계좌 상세의 과한 설명과 거친 UX를 정리한다.

## Scope

### 1. 공용 숫자 입력

- 기존 `NumericInputField`를 숫자 키패드 + 천 단위 포맷 표시 방식으로 바꾼다.
- 외부 상태에는 숫자 문자열만 올리고, 화면에는 포맷된 문자열을 보여준다.
- 기존 금액 입력 화면은 새 공용 입력을 그대로 사용한다.

### 2. 1~28일 선택

- 자동이체일, 납부일 같은 `1~28일` 값은 자유 숫자 입력 대신 select box 방식으로 바꾼다.
- 현재 스택에 별도 picker 라이브러리가 없으므로, 공용 `DayOfMonthSelectField`를 만든다.
- 필드를 누르면 인라인 선택 목록이 열리고, 선택한 날짜가 값으로 반영된다.

### 3. 알림 화면

- 상단 요약 카드 3개를 제거한다.
- `전체`, `읽지 않음`, `안내`, `공지` 필터 칩에 빨간 원형 배지를 붙인다.
- 배지는 개수가 있을 때만 표시한다.

### 4. 설정 화면

- 상단 요약 카드를 제거한다.
- `빠른 이동`, `계정 액션`, `여기에서만`, `세션 종료` 같은 설명성 카피를 없앤다.
- 행 높이, 패딩, 아이콘 원 배지 크기, 카드 간격을 하나의 규칙으로 통일한다.

### 5. 금액 표시와 복사

- 계좌 상세 상단 잔액은 `금액을 확인하려면 꾹 누르세요.` 문구를 좌측 정렬로 보여준다.
- `press in / press out` 동안만 실제 금액이 표시되게 바꾼다.
- 계좌번호가 보이는 곳에는 텍스트 바로 옆 복사 아이콘을 붙인다.

## Testing

- 공용 숫자 입력: 포맷 표시와 raw value 반환을 unit test로 잠근다.
- 일자 선택: 1~28 선택값 반영을 unit test로 잠근다.
- 알림: 요약 카드 제거와 배지 표시를 unit test로 잠근다.
- 설정: 과한 카피 제거와 행 통일 구조를 unit test로 잠근다.
- 상세 히어로: 길게 누르는 동안만 금액이 보이는 동작을 unit test로 잠근다.
- 전체 회귀는 `pnpm test --runInBand`, `pnpm run ci:web`, `pnpm test:e2e`로 확인한다.
