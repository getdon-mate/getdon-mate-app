# Visual Completeness Design

**Date:** 2026-03-15

## Goal

기능 범위를 늘리지 않고 `통계`, `캘린더`, `시작 화면`, `회비 상태`의 시각 완성도를 올린다. 방향은 장식을 늘리는 것이 아니라 정보 위계를 더 또렷하게 만들고, 작은 화면에서도 차분하고 성숙하게 보이도록 정제하는 것이다.

## Direction

- 과한 그라데이션, 강한 모션, 과장된 장식은 쓰지 않는다.
- 대비, 타이포, 여백, 상태 톤만으로 화면의 집중점을 만든다.
- 작은 화면에서 카드 높이와 줄 수를 줄이되, 핵심 수치와 상태는 더 빨리 읽히게 한다.
- 기존 홈/상세 화면과 어긋나지 않도록 같은 라운드, 보더, 중립색 계열을 유지한다.

## Scope

### 1. Statistics

- 상단 요약 영역을 더 분명한 한 덩어리로 보이게 조정한다.
- `월별 추이`와 `카테고리 비중` 사이의 계층을 더 벌려서 먼저 볼 정보와 보조 정보를 구분한다.
- 선택된 카테고리나 강조 수치를 카드 형태로 더 또렷하게 보여준다.
- 막대/트랙의 톤을 단순화해서 라벨과 값이 먼저 읽히게 만든다.

### 2. Calendar

- 월 헤더와 월 이동 액션을 더 단정하게 정렬한다.
- 선택된 날짜, 오늘, 일정이 있는 날짜의 표현을 더 분명하게 나눈다.
- 선택한 날짜의 일정 패널을 달력 아래에서 가장 먼저 읽히는 블록으로 만든다.
- 빠른 액션은 남기되, 일정 본문보다 시선을 덜 가져가게 톤을 낮춘다.

### 3. Splash

- `getdon mate` 워드마크의 타이포 위계를 정리한다.
- 설명 문구를 줄이고 제품 준비 상태를 더 짧고 단정하게 보여준다.
- 로딩 점은 장식이 아니라 리듬만 주는 수준으로 유지한다.

### 4. Dues

- 미납/완료 상태 배지와 요약 카드를 더 안정적으로 읽히게 만든다.
- 리마인드 기록과 보조 설명의 대비를 줄여서 주요 금액/상태가 먼저 보이게 한다.
- 카드 내부 여백과 줄 수를 줄여 작은 화면에서 화면 체류 높이를 낮춘다.

## Testing

- 시각 위계 변경으로 함께 바뀌는 라벨/상태 문구는 unit test에 반영한다.
- 전체 회귀는 `pnpm test --runInBand`, `pnpm run ci:web`, `pnpm test:e2e`로 잠근다.
- e2e는 기존 흐름 유지에 집중하고, 새 상호작용이 생기지 않으면 시나리오 회귀만 확인한다.
