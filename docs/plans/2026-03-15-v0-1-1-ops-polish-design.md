# Getdon Mate v0.1.1 Operational Polish Design

## Goal

방금 추가한 협업 기능들을 실제 운영 도구처럼 연결하고 다듬는다. 신규 기능 수를 늘리기보다, 리마인드/알림/캘린더/게시판/통계가 서로 자연스럽게 이어지고 모바일에서도 덜 답답하게 느껴지도록 polish한다.

## Scope

- 리마인드 액션의 최근 이력과 알림 반영을 더 분명하게 노출
- 캘린더에서 날짜 선택과 일정 집중 흐름 추가
- 게시판 작성/댓글 흐름과 빈 상태를 더 간결하게 정리
- 통계 탭 핵심 수치 가독성 보강
- 모바일 밀도 기준으로 카드/입력/CTA 레이아웃 축소
- 운영 흐름 기준 e2e 회귀 추가

## Product Direction

“쓸 수 있는 도구” 느낌을 더 강하게 만든다. 각 탭이 제각각 기능을 나열하는 대신, 최근 활동과 다음 행동이 자연스럽게 이어지도록 정리한다. 시각 장식은 늘리지 않고, 정보 우선순위와 레이아웃 압축으로 완성도를 높인다.

## UX Decisions

### 1. Reminder Activity

- `납부 안내`와 `송금 요청`을 보낸 뒤 끝내지 않고, 최근 보낸 이력을 같은 화면에서 다시 확인할 수 있게 한다.
- 미납 멤버 카드에는 “마지막 안내 시각/종류”를 짧게 보여준다.
- 캘린더 일정에도 리마인드 활동이 포함되도록 한다.

### 2. Calendar Focus

- 월간 그리드에서 날짜를 누르면 아래 리스트가 해당 날짜 일정 중심으로 정렬된다.
- 선택된 날짜가 없으면 최신 활동 날짜를 기본 선택한다.
- 일정 tone에 따라 작은 배지/포인트 색을 달리해 회비, 거래, 공지를 구분한다.

### 3. Board Flow

- 작성 폼은 접힘 기본값이 아니라 유지하되, 세로 길이가 과하게 늘지 않게 압축한다.
- 공지는 별도 시각 표시를 두고, 댓글 입력은 카드 하단 한 줄 흐름으로 정리한다.
- 첫 글 전용 빈 상태는 안내보다 작성 유도를 우선한다.

### 4. Statistics

- 월별 추이 위에 “이번 달 순유입 / 총 출금 / 미납 인원” 요약을 추가한다.
- 차트 자체는 단순 막대 유지, 대신 수치 카드로 빠른 읽기를 돕는다.

### 5. Mobile Density

- 작은 화면에서는 summary card를 세로 스택으로 바꾸고, 액션 버튼 최소 너비를 줄인다.
- 게시판/거래 필터/댓글 입력의 세로 패딩을 약간 줄여 스크롤 부담을 덜어준다.
- 상세 탭 바는 유지하되, 각 탭 pill 폭을 더 작게 맞춘다.

## Architecture

- `GroupAccount.reminders`를 그대로 사용하되, selector에서 최근 멤버별 리마인드와 캘린더 이벤트를 파생한다.
- 캘린더는 `selectedDate` 로컬 상태를 추가해 그리드와 목록을 연결한다.
- 게시판/통계는 새 상태를 만들지 않고 현재 데이터를 재배치하는 수준으로 제한한다.

## Error Handling

- 리마인드/댓글 액션은 실패 시 alert, 성공 시 toast를 유지한다.
- 선택 날짜에 일정이 없을 경우 빈 날짜 상태를 보여주되, 전체 일정 목록으로 쉽게 복귀할 수 있게 한다.

## Testing

- 단위 테스트: 최근 리마인드 표시, 캘린더 날짜 선택, 통계 요약 카드, 게시판 폼/댓글 입력
- e2e: 리마인드 후 알림센터 진입, 캘린더 날짜 선택, 게시판 작성 후 댓글, 거래 필터 열고 닫기
- 최종 검증: `pnpm test --runInBand`, `pnpm run ci:web`, `pnpm test:e2e`
